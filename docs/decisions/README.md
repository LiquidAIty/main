# Architecture Decisions (ADR)

Use this folder for concise Architecture Decision Records.

## Naming
- `YYYY-MM-DD-short-title.md`
- Example: `2026-05-24-single-runtime-env.md`

## Template

```md
# ADR: <title>

## Status
Accepted | Superseded | Proposed

## Context
What problem forced this decision?

## Decision
What was chosen?

## Consequences
What improves? What tradeoffs exist?

## Verification
Which tests/smokes prove this is working?
```
