# Skill Packet Fable Handoff Spec

> Transition policy: this is a legacy/source handoff model. Relevant skill memory now contributes
> to the Context Packet and one active CoderPacket/spec-as-prompt.

## Purpose

Every real Fable implementation attempt begins with relevant KnowGraph skill memory. The scout or
planner retrieves a compact skill packet before writing the bounded attempt, embeds it in the
Fable prompt, and Fable obeys its guardrails, reuses its query patterns, avoids its failed
attempts, and writes results back so the next task starts smarter.

## Handoff Rule

A real code or task prompt may not become a Fable attempt without a Skill Memory Packet. The
packet is generated by the existing deterministic retrieval command, never by an LLM:

```powershell
py -3.12 services/knowgraph/skill_ingest.py packet --prompt "<task prompt>" --limit 3 --json
```

Optional exact selectors (`--skill-id`, `--spec specs/<name>.md`) may be added when known. An
empty packet does not block the handoff; it triggers the existing repo law:
`No matching skill found; successful completion must create a new skill.`

## Loop

1. User asks for work.
2. Scout reads `AGENTS.md`, `PLAN.md`, and relevant specs.
3. Scout refreshes or proves fresh CBM.
4. Scout runs skill packet retrieval with the task prompt.
5. Scout writes the bounded Fable prompt embedding the packet (see contract below).
6. Fable works inside the bounded attempt.
7. Fable updates the matching skill file with `@attempt_result`, proof, or failure plus guardrail.
8. Skill files re-ingest: `py -3.12 services/knowgraph/skill_ingest.py ingest --repo-root .`
9. Next task starts smarter.

## Packet JSON Shape

As produced by `packet --json` (see `specs/knowgraph-skill-retrieval-spec.md`):

```text
{ packet_version, query: {skill_id, spec, prompt, limit},
  skills: [ { skill_id, score, match_reasons: [{kind, weight, evidence}],
              status, source_path, applies_to, requires, related_to,
              guardrails, decisions, failed_attempts, query_patterns,
              attempts: [{id, status, result_status, proof_claims, validations}],
              sections: [{heading, text}] } ] }
```

Output is deterministic for identical graph state and arguments.

## Fable Prompt Contract

The Fable prompt must contain, in order:

1. `## Task Prompt` — the bounded task statement.
2. `## Source Spec` — the spec path, or `none declared`.
3. `## Skill Memory Packet` — the generating command line and the packet JSON verbatim in a fenced
   block. If the packet matched no skills, the section instead carries the new-skill rule line.
4. `## Code Evidence Packet` — the scout-composed fresh code evidence JSON
   (`specs/codegraph-context-reader-spec.md`), attached via `--code-evidence <packet.json>`; when
   absent, an explicit placeholder tells the scout to attach one and obligates Fable to gather
   current code evidence itself via fresh CBM.
5. `## Required Behavior` — fixed obligations:
   * refresh or prove fresh CBM before code work and record counts;
   * treat every packet guardrail as a hard constraint;
   * do not retry packet failed attempts except along their recorded retry direction;
   * use packet query patterns plus fresh CBM/CodeGraph queries for current code evidence;
   * do not relitigate packet decisions and rejected alternatives without new evidence;
   * run packet validations and spec proof commands honestly, never faking success;
   * append the bounded attempt and its `@attempt_result` (status, `@cbm_after`, proof) to the
     matching `skills/*.md` file, or create the smallest useful new skill;
   * re-ingest skills after closeout;
   * stop after reporting; no commit or push unless explicitly instructed.

The helper command renders this contract deterministically:

```powershell
py -3.12 services/knowgraph/skill_ingest.py handoff --prompt "<task prompt>" [--spec specs/<name>.md] [--limit 3]
```

## Relationship To CodeTaskPacket

`specs/code-task-packet.md` owns the full execution packet (allowed files, intended delta, stop
conditions). The Skill Memory Packet is one required input to building that packet and to any
interim Markdown Fable prompt. This spec does not change the CodeTaskPacket schema.

## Not Included Yet

* UI or backend route
* automatic re-ingest watcher
* LLM GraphRAG, Text2Cypher, vector search
* LocalScout implementation

## Acceptance

* clear rule: every real Fable implementation attempt includes a skill packet
* packet generated by the existing retrieval command
* Fable prompt template includes a `Skill Memory Packet` section
* template obligates guardrails, query patterns, write-back, and re-ingest
* `handoff` output is deterministic; tests pass; retrieval stays read-only

## Validation

```powershell
py -3.12 -m unittest discover -s services/knowgraph -p "test_skill*.py" -v
py -3.12 services/knowgraph/skill_ingest.py handoff --prompt "Neo4j skill ingestion guardrails" --limit 3
```
